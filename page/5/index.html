【异常-SpringBoot-1】java.net.BindException_Address already in use_bind

Posted on 2020-10-17

错误

原因

项目所使用的端口已经被其他进程占用。

本项目所使用的端口是8080。

解决

在终端使用netstat -aon|findstr 8080命令查看哪些进程占用了8080端口。

使用taskkill /pid 38500 /f命令终止进程对端口的占用。

注意38500是指被占用的进程PID。

再次执行netstat -aon|findstr 8080命令查看，就没有占用进程了。

【异常-Tomcat-2】org.apache.tomcat.util.bcel.classfile.ClassFormatException_Invalid byte tag in constant pool_19

Posted on 2020-10-16 Edited on 2020-10-17

错误

原因

有的依赖包依赖JDK1.9版本的语法，tomcat8.x解析不了。

springboot项目的jackson系列的包版本冲突了。

解决

有两个解决方式：

1.提高tomcat的版本到tomcat9.x

2.找到发生冲突的jar包，降低它的版本。

本次的解决方式是通过将tomcat版本提高到了tomcat9，就没有发生上述报错了。

同时注意观察报错给出的日志，就能发现是哪些包发生了冲突。

【异常-IDEA-2】the output path is not specified for module "xxx".

Posted on 2020-10-16

错误

当从GitHub上克隆导入项目后，运行报错

原因

这个错误是因为没有设置output的路径。

解决

第一步：在Modules设置里勾选”Inherit project compile path”

第二步：设置Project中的”Project compiler output” ，选择”Project的路径”+”\out”。
